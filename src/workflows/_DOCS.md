## Relevant File Locations

- `src/workflows/registration.py`
- `src/workflows/registration_workflow.py`
- `src/workflows/assignment_feedback_workflow.py`
- `src/workflows/parsing_utils.py`
- `src/main.py`

## Runtime Entry Flow

- `main.build_ducks(...)` instantiates workflow-style ducks based on duck type:
  - `registration` -> `RegistrationWorkflow`
  - `assignment_feedback` -> `AssignmentFeedbackWorkflow`
- `DuckOrchestrator` invokes these via the shared `DuckConversation` interface (`__call__(DuckContext)`).

## Registration Workflow

- `Registration` handles the step-by-step user flow:
  - net ID capture and validation
  - email token verification
  - nickname assignment (with suspicious-name checks)
  - role selection/assignment
- `RegistrationWorkflow` runs `Registration.run(...)` and optionally delegates completion to a configured `registration_bot` tool.
- Registration progress text is generated by `describe_registration_progress(...)`.

## Assignment Feedback Workflow

- `AssignmentFeedbackWorkflow` guides rubric-based grading for uploaded markdown reports.
- Core stages:
  - collect report upload
  - identify project name (header match, then fallback scanner agent)
  - load rubric/rules from YAML files
  - grade rubric items via single-item grader agent
  - format and send markdown results
- `parsing_utils.py` provides report/rubric flattening and markdown formatting helpers used in grading output flow.
