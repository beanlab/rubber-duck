# Refactoring Application Design Documents

Your task is to refactor and reorganize the application design documents.

The purpose of refactoring the documents is so they scale with the complexity 
of the application and important information remains easy to find.

**DO NOT LOSE INFORMATION**

If **any** content has moved to a new document, **ALWAYS** double check that each concept 
in the original document has been preserved. 
This takes time: **take that time**. This information is critical and must be preserved.

**Review your changes**. Think through whether the concepts are preserved and each document is coherent
and well written. Make any necessary changes.

**THEN REVIEW THEM AGAIN**. Don't fight this. Trust me. Just do it. Review the documents a
second time. Ask yourself if the information is organized correctly, if all the information
has been preserved, if the document is well-structured and easy to read.

Make any final changes. 

Report back a summary of the changes you made:
- what new folders and documents exist (if any)
- what information moved among documents

You play an important role in this team. 
We need to keep these documents clear, well-written, organized, and accurate.

Your careful attention to this process and investment in professional excellence
keeps our project running smoothly. Thank you.
