## Shared development harness and workflow design in `myteam`

### Summary

Much of the discussion focused on the evolving development
harness in `myteam`. The goal is to standardize development
support across `myteam`, Rubber Duck, x-campus, Markdown Data,
and related projects. The desired harness separates three
concerns:

- philosophy and design approach
- application documentation and agent legibility
- the process the agent follows to implement code changes

The team discussed a lightweight workflow mechanism that chains
terminal agent sessions together. The current design injects a
step objective into a Codex session, watches for a structured
JSON result, closes the session, and starts the next step. This
approach aims to stay simple, reuse the existing terminal
experience, and support multiple agent clients such as Codex and
Claude with minimal adapter code.

The group also discussed where workflows are still incomplete:
initial prompt timing, workflow resumption, reusing prior
sessions, skipping to a later step safely, and handling larger
features through milestones rather than a single fixed pipeline.

The desired feature-development process is more rigorous than the
current state. It should include interface design, application
document updates, framework-level refactoring when needed,
feature design against the framework, test-first implementation,
independent review, and final release hygiene such as changelog
and version checks.

### Decisions

- `project-management` should be treated as the parent skill
  rather than importing only backlog and meeting-minutes as peer
  folders.
- Application documentation should remain separate from backlog
  and other project-management concerns.
- Simple workflows may remain YAML-based; more complex logic can
  move into Python.
- A single shared dev harness in `rosters` is the target
  direction.
- Near term, the team will bootstrap the harness while learning
  from small feature work.

- [Ty] Explore support for workflow resume, step skipping, and

- [Team] Develop a workflow or skill structure for the desired
