# Rubber Duck Dev Team Minutes

Date: 2026-04-08 10:57 America/Denver

## Summary

The team reviewed progress on meeting-minutes download support,
workflow-driven agent harnesses, Teams integration, stats prompt
changes, and possible course-specific RAG. A major theme was the split
between highly autonomous agent workflows for standard software stacks
and human-driven workflows for creative or non-standard projects.

## Main Points

- `myteam` meeting-minutes download is now working and can be rolled
  out across courses.
- The team sees two agent-use modes:
  standardized autonomous delivery and human-driven augmentation.
- Rubber Duck and related tooling are better aligned with the second
  mode, where humans stay in control and agents reduce friction.
- A staged design-first workflow worked much better than an earlier
  autonomous implementation attempt.
- Planned workflow support will likely use YAML-defined stages and
  bounded agent runs with controlled context handoff.
- Workflow UX may become a separate `myteam` entry point.
- Teams integration is blocked by Azure and app-permission setup, not
  by a core code limitation.
- Stats prompt changes using user-led conversation plus
  `conclude_conversation` are working well, but early closure still
  needs monitoring.
- OpenAI service instability may need targeted retry handling and
  better admin visibility.
- Course-specific RAG remains a promising future direction.
- Team structure may need a senior-dev layer to scale work without
  overloading Professor Bean.

## Decisions

- Continue rolling out meeting-minutes downloads to courses.
- Continue exploring workflow architecture inside `myteam`.
- Keep the user-led stats conversation model with explicit conclude
  behavior.
- Investigate retry behavior around upstream OpenAI failures before
  adding more logic.
- Consider a more layered team structure for spring and summer work.
